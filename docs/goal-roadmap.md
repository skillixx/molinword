# AI Word 文档助手 Goal 阶段规划

## Goal 0：本地开发基线

### 阶段目标

建立一个可以本地运行、可以展示产品雏形的基础工程。

### 当前状态

已完成。

### 已交付内容

- Vite + React + TypeScript 前端工程
- 本地 Node + Express API 代理
- 工作台页面
- 文档编辑页面
- 模拟 AI 大纲生成
- 模拟 AI 润色
- 墨灵 Token 网关环境变量预留
- Vite `/api` 代理配置

### 验收标准

- 可以访问 `http://127.0.0.1:5188`
- 可以看到 AI Word 文档助手页面
- 点击“生成大纲”可以进入编辑器
- 点击“润色当前内容”可以得到模拟结果
- `npm run build` 构建通过

## Goal 1：真实 AI 能力接入

### 阶段目标

让产品从模拟 AI 切换为真实调用墨灵 Token 网关。

### 主要任务

- 配置 `.env`
- 接入真实 `MOLIN_GATEWAY_API_KEY`
- 调通墨灵 Token 网关 `/v1/chat/completions`
- 完善生成大纲接口
- 新增生成正文接口
- 完善润色接口
- 增加加载状态
- 增加错误提示
- 增加 AI 返回内容格式校验

### 交付物

- 真实 AI 生成大纲
- 真实 AI 生成正文
- 真实 AI 润色
- 前端显示当前使用“墨灵 Token 网关”
- AI 请求失败时自动降级或提示

### 验收标准

- `/api/health` 返回 `gatewayConfigured=true`
- 生成大纲结果来自真实模型
- 润色结果来自真实模型
- 浏览器源码和网络请求中不暴露 `sk`
- 网关异常时页面不会崩溃

## Goal 2：文档编辑体验完善

### 阶段目标

把当前 textarea 编辑器升级为更接近 Word 的编辑体验。

### 主要任务

- 接入富文本编辑器，建议使用 Tiptap
- 支持标题、正文、加粗、列表
- 支持大纲识别
- 支持点击大纲定位章节
- 支持选中文字后调用 AI
- 支持替换原文、插入下方、复制结果
- 增加快捷操作菜单

### 交付物

- 可用的富文本编辑器
- 左侧文档大纲
- 右侧 AI 助手
- 选中文本 AI 操作

### 验收标准

- 可以编辑标题和正文
- 可以对选中文字进行润色
- AI 结果可以替换原文
- 文档布局在桌面端显示稳定
- 基础移动端不出现明显重叠

## Goal 3：MySQL 文档存储

### 阶段目标

让用户创建和编辑的文档可以保存、再次打开和管理。

### 当前状态

已完成核心业务接入。

当前实现采用 Node.js + `mysql2` 直接访问 MySQL，Prisma 暂未接入；第一版先保证文档存储主流程稳定。

### 主要任务

- 接入 MySQL
- 接入 Prisma
- 设计 `documents` 表
- 设计 `document_versions` 表
- 设计 `ai_request_logs` 表
- 实现创建文档
- 实现保存文档
- 实现自动保存
- 实现最近文档列表
- 实现文档重命名、删除、复制

### 交付物

- 文档可以保存到 MySQL
- 工作台展示真实最近文档
- 编辑器可以打开历史文档
- AI 请求有基础日志记录

### 已交付内容

- 后端文档 CRUD 接口
- 创建文档保存到 `documents`
- 编辑器手动保存
- 编辑器自动保存
- 最近文档读取真实 MySQL 数据
- 文档重命名、删除、复制
- 手动保存版本写入 `document_versions`
- AI 请求日志写入 `ai_request_logs`

### 验收标准

- 刷新页面后文档不丢失
- 最近文档来自数据库
- 自动保存不会频繁打爆接口
- 删除文档有确认逻辑
- 数据表字段能支持后续绑定墨灵 `user_id`

## Goal 4：Word 导出与 MinIO 存储

### 阶段目标

实现从在线文档导出 `.docx`，并把导出文件存储到 MinIO。

### 当前状态

已完成核心业务接入。

### 主要任务

- 接入 `docx` 库
- 接入 MinIO SDK
- 设计 `files` 表
- 实现基础 Word 导出
- 支持标题、段落、列表
- 支持基础页面设置
- 导出文件上传 MinIO
- 返回下载链接

### 交付物

- `.docx` 导出接口
- MinIO 文件存储
- 文件记录入库
- 前端“导出 Word”按钮可用

### 已交付内容

- 接入 `docx` 生成 `.docx`
- 解析编辑器 HTML，导出标题、段落、列表基础结构
- 上传导出文件到 MinIO bucket：`moling-word`
- 导出文件索引写入 `files`
- 新增本地下载接口 `/api/files/:id/download`
- 前端“导出 Word”按钮接入真实接口
- 导出前自动保存一个文档版本

### 验收标准

- 点击“导出 Word”可以生成 `.docx`
- 下载后的 Word 能正常打开
- 标题和正文格式清晰
- 导出文件能在 MinIO 中找到
- 数据库有文件索引记录

## Goal 5：墨灵平台 SSO 接入

### 阶段目标

让用户可以从墨灵平台进入 AI Word 文档助手，并完成身份打通。

### 主要任务

- 新增 `/molin/launch` 页面或路由
- 支持接收 `?ticket=lt_xxx`
- 后端调用墨灵平台 `POST /api/internal/app-launch/verify`
- 获取 `user_id/app_id/product_id`
- 建立本应用本地 session
- 所有文档数据绑定 `user_id`
- 增加未登录/票据无效处理

### 交付物

- 墨灵平台入口路由
- SSO ticket 校验
- 本地登录态
- 用户文档隔离

### 验收标准

- 从墨灵平台点击“进入应用”可以打开本应用
- ticket 有效时可以进入工作台
- ticket 过期或重复使用时给出提示
- 不同 `user_id` 只能看到自己的文档
- 本地开发模式仍可使用模拟用户

## Goal 6：墨灵平台计费接入

### 阶段目标

核心 AI 功能接入墨灵平台计费能力。

### 建议 usage_type

```text
word_outline_generate
word_body_generate
word_polish
word_export_docx
```

### 主要任务

- 明确采用后付钱包还是预付积分
- 封装 `BillingService`
- 每个计费动作生成幂等键
- 后付调用 `POST /api/internal/product-usage-events`
- 预付调用 `POST /api/internal/entitlement-consume`
- 高成本任务预留 `reserve -> settle / release`
- 增加余额不足/额度不足提示
- 记录计费结果

### 交付物

- AI 大纲生成计费
- AI 正文生成计费
- AI 润色计费
- Word 导出计费，可选
- 计费失败提示

### 验收标准

- 同一个幂等键不会重复扣费
- 余额不足返回明确提示
- 额度不足返回明确提示
- 计费成功后平台能查到消费记录
- AI 调用失败时不会错误扣费

## 推荐推进顺序

建议按以下顺序推进：

1. Goal 1：真实 AI 能力接入
2. Goal 2：文档编辑体验完善
3. Goal 3：MySQL 文档存储
4. Goal 4：Word 导出与 MinIO 存储
5. Goal 5：墨灵平台 SSO 接入
6. Goal 6：墨灵平台计费接入
7. Goal 7：产品体验优化

## 当前最应该做的下一个 Goal

```text
Goal 5：墨灵平台 SSO 接入
```
