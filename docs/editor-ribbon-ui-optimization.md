# 编辑器功能区 UI 五轮优化记录

## 目标

将在线编辑器功能区调整为紧凑、清晰的 WPS 风格分组工具栏，并确保桌面端和移动端均不需要横向滑动选择命令。

## 五轮优化

### 第一轮：建立基线

- 在 `1600 × 1000` 窗口、中央编辑区宽约 `723px` 时测量功能区。
- 初始“开始”功能区高度约 `317px`，虽然没有横向滚动，但正文被明显下压。
- 结论：优先处理按钮高度、表单默认内边距和分组换行策略。

### 第二轮：压缩控件密度

- 功能按钮统一为 `30px` 高度。
- 收紧按钮内边距、图标间距、分组间距和下拉框高度。
- 修复全局 `label/select/input` 样式对功能区控件造成的额外外边距和内边距。

### 第三轮：重建视觉层级

- 将撤销、重做、斜体、链接、上标、下标等命令改为统一方形图标按钮。
- 分组采用轻量卡片和明确的底部组名。
- 使用十二列栅格与显式跨度类固定各业务分组宽度，避免布局依赖 JSX 顺序。
- 实测“格式”功能区高度约 `140px`，“开始”功能区约 `200px`。

### 第四轮：完善交互反馈

- 增加悬停、按下、选中、禁用和键盘焦点状态。
- 激活按钮使用更明确的墨绿色底色和边框。
- 下拉框和按钮共享一致的焦点环，支持键盘访问。

### 第五轮：上下文降噪与响应式收口

- 表格编辑命令只在光标位于表格中时展示；默认只显示“插入表格”和上下文提示。
- `390px` 移动端将六个页签排成不超过两行，功能分组使用两列栅格，宽分组自动占满整行。
- 自动化测试逐一切换桌面端与移动端六个页签，断言每个功能区 `scrollWidth === clientWidth`。

## 后续建议

1. 后续新增高复杂度命令时，优先使用上下文工具组，不要持续扩大常驻功能区。
2. 常用命令保持文字加图标；低频且语义明确的命令使用方形图标，并保留 `title` 与 `aria-label`。
3. 新分组必须声明 `ribbon-span-*` 跨度类，并同步补充桌面端、移动端无溢出测试。
4. 如果未来命令数量继续增长，可增加“自定义常用工具”能力，但不建议恢复横向滚动。
