# 前端开发「完成」的定义（Definition of Done）

> 适用：`web/admin-console`、`web/user-console` 全部前端页面交付（Codex / 前端工程师甲乙）。
> 目的：让「前端开发完成」对账的是**契约清单逐项有页面且过验收**，而不是「我感觉做完了」。
> 背景：曾出现 Codex 报「前端完成」但实际漏项——后端 PR #273 给模型目录加了 `visible_scope` 定向可见性，
> 该功能合并在 Codex 报「完成」之后，前端没跟上（`TokenModelListView.vue` 里 `grep visible_scope` 零命中）。
> 根因是缺一道「拿最新契约对账」的关卡。本文档把这道关卡固化下来。

---

## 核心原则

**「完成」= 契约清单逐项有页面 + 五关全过 + 无未对接的后端 delta。**

只要存在「已合并到 main、但前端未对接的后端接口/字段变更」，就**不能**说「前端开发完成」，
只能说「截至 commit X 的范围完成」。

---

## 每个前端任务的五道关卡（全过才算「完成」）

### 关卡 0 — 契约对账（防 delta 漏接，这是最容易缺的一关）
- [ ] 以 `docs/frontend-api-reference.md §14` + 对应 `docs/frontend-task-*.md` 为**唯一清单**，逐个接口/字段核对「是否有对应 UI」。
- [ ] **拉最新 main 再对账**：`git log --oneline <上次完成点>..main`，凡近期合并的后端 PR 新增/改了接口字段的，确认前端已跟上。
- [ ] 任务单里每个字段都能在页面代码里 `grep` 到（命中=已接，零命中=漏接）。
- [ ] 契约若与实现冲突，**只列出需后端补充/澄清的点，不自行实现后端**（分工红线）。

### 关卡 1 — 构建
- [ ] `npm run build` 通过（涉及的 admin-console / user-console 两端，对应 CI「管理后台构建 / 用户控制台构建」）。

### 关卡 2 — 契约一致性（QA L2）
- [ ] 前端 api 封装的**路径 / 方法 / 请求体 / 响应字段 / 分页 / 错误码**与 §14 逐项一致。
- [ ] 列表分页一律扁平 `{items,page,page_size,total}`（D-95），禁止 `{list,pagination}`。
- [ ] 批量写入 body 统一 `items` 键。
- [ ] 字段名严格对齐后端 DTO（「路径/字段未对齐前端」是本项目反复出现的根因）。

### 关卡 3 — 交互完整性自检（每个页面）
- [ ] 五态齐全：加载中 / 空数据 / 错误（透传后端 message）/ 无权限降级 / 正常。
- [ ] 表单：必填校验、编辑回填、提交语义（如 PATCH 整体覆盖）正确。
- [ ] 次级接口的权限依赖缺失时**降级提示而非崩页**（例：模型可见性表单的分组/角色下拉需 `group:manage`/`role:manage`，缺权限时降级）。
- [ ] 计费红线：Agent / Skill / 插件 / 角色全部免费，UI 不得出现对它们收费的文案；计费仅出现在模型 token 调用。

### 关卡 4 — PM 业务验收（终验）
- [ ] 派产品经理按任务单「验收」一节逐条走，业务规则正确、无 P0~P2 遗留。
- [ ] 条件允许时补 L3 真浏览器 E2E 冒烟（项目既有四级阶梯：L1 构建 / L2 契约 / L3 E2E / PM 验收）。

---

## 「整体前端完成」的判定（项目级）

同时满足：
1. **所有** `frontend-task-*.md` 列出的页面都已实现且过关卡 1–4；
2. `§14` **每个接口**都有 UI 消费（无「接口就绪但前端没接」）；
3. `git log` 里**没有**「已合并但前端未对接的后端 delta」；
4. PM 出具验收通过结论。

不满足第 3 条时，措辞必须是「截至 commit X 的范围完成」，不能是「前端开发完成」。

---

## Codex 报「完成」时应附的最小证据

为避免空口「完成」，报完成时随附：
- 对账依据：对的是哪些 `frontend-task-*.md` / `§14` 小节，截止到哪个 main commit。
- 关卡 1 结果：两端 `npm run build` 通过。
- 关卡 0 抽查：对近期后端 PR 的 delta 已逐项确认（列出 PR 号 + 对接页面）。
- 已知缺口（如有）：明确列出「未对接项 + 原因」，而非笼统说「完成」。
