# 前端对接任务：分组管理新增「组角色」

> 负责人：前端工程师甲（web/admin-console）
> 对应后端：PR #186「组绑定全局角色 group_roles」（已合并 main、已部署测试环境）
> 接口契约权威：`docs/full-api-design.md` 3.17.4a、`docs/api-test-guide-backend-a.md` 9.13a

## 一、背景（了解即可，不需实现后端逻辑）

后端新增「组绑定全局角色」：给一个分组绑定若干全局角色后，**该组成员会自动继承这些角色**，用于商品访问/定价。管理后台需要在「用户分组管理」页里增加一块「组角色」的配置入口——和现有的「组权限」「邀请码」是同级的子功能。

> 前端只负责调这 3 个接口做增删查，**落组继承、商品可见性都是后端行为，前端不实现**。

## 二、接口契约（管理端，需管理员登录 + 双重认证，权限 `group:manage`）

与现有「组权限」接口完全同构，照搬即可。

### 1. 列出组绑定的角色
`GET /api/admin/user-groups/{id}/roles`

返回 `data`（数组，非分页）：
```json
[
  { "id": 1, "group_id": 3, "role_id": 5, "created_at": "2026-06-19T18:00:00Z" }
]
```
> 注意：只返回 `role_id`，**不含角色名**。前端需用已有的「角色列表」接口（`GET /api/admin/roles`）做 id→名称 映射展示（角色管理页应已加载过角色列表，可复用）。

### 2. 给组绑定角色
`POST /api/admin/user-groups/{id}/roles`
请求体：
```json
{ "role_id": 5 }
```
成功：HTTP `201`，`data: null`。

### 3. 解绑
`DELETE /api/admin/user-groups/{id}/roles/{role_id}`
成功：HTTP `200`，`data: null`。

### 错误码（需在 UI 给出提示）

| HTTP | code | 场景 | 建议提示 |
|---|---|---:|---|
| 400 | 40000 | role_id 为空 / 角色不存在 / **绑定系统角色（admin）被拒** | 按 message 原文提示（如「不允许将系统角色绑定到分组」） |
| 404 | 40400 | 分组不存在 / 该角色未绑定到此分组 | 「分组不存在」/「该角色未绑定」 |
| 409 | 40900 | 该角色已绑定到此分组（重复） | 「该角色已绑定」 |

## 三、前端需要做的（web/admin-console）

1. **API 封装**：`src/api/group.ts` 增加 `listGroupRoles(id)` / `addGroupRole(id, roleId)` / `removeGroupRole(id, roleId)` 三个方法（照搬同文件里 `组权限` 的写法）。
2. **页面**：在用户分组管理页（`src/views/group/UserGroupListView.vue` 或其分组详情/抽屉里）增加「组角色」区块，与「组权限」「邀请码」并列：
   - 列表展示该组已绑定的角色（用 `GET /api/admin/roles` 映射出角色名展示，不要只显示 id）；
   - 一个「添加角色」入口：从全局角色列表里选一个角色 → 调 `addGroupRole`；
   - 每条绑定可「解绑」→ 调 `removeGroupRole`；
   - 操作成功后刷新列表，失败按上表错误码提示。
3. 角色选择器建议**过滤掉 `admin` 等系统角色**（绑定会被后端拒 400），或保留但捕获 400 友好提示。

## 四、不需要处理的事项（避免过度实现）

- ❌ 不要实现「绑了角色后用户能看到哪些商品」的展示——那是商品侧（用户端）的事，本页不涉及。
- ❌ 不要前端校验角色能不能绑（系统角色拦截由后端做，前端按 400 提示即可）。
- ❌ 不要把「组角色」和「组内身份」（成员的 admin/member，那是 `members` 接口的 `group_role` 字段）搞混——两者是不同概念：
  - **组角色**（本任务）= 绑定的全局角色，影响商品访问；
  - **组内身份** = 成员在组里是管理员还是普通成员，已有功能，不要动。

## 五、验收口径

- 在某分组「组角色」区块添加一个角色 → 列表出现该角色（显示角色名）。
- 重复添加同一角色 → 提示「该角色已绑定」（409）。
- 尝试添加 admin（若没过滤）→ 提示「不允许将系统角色绑定到分组」（400）。
- 解绑 → 列表移除。

## 六、范围边界

- 只动 **web/admin-console** 分组管理相关的 api 封装与页面。
- 不碰后端、不碰数据库、不碰用户控制台（web/user-console）。
- 「组内身份/成员管理/组权限/邀请码」等既有功能不要改动。
